# Render Checklist · Phase B 渲染验收清单

用于替代外部 checklist。进入交付前，至少完成一次全量自检。

---

## P0 · 必过项

- [ ] 页面顺序符合 `render-plan.json`，没有缺页、重复页、错位页
- [ ] 封面、四幕章节页、结束页都存在
- [ ] 每页只有一个明确主信息点，没有把两页内容硬塞成一页
- [ ] 所有基础 layout 和 extra layout 都继承同一套主题色 token（`--ink`、`--paper`、`--accent`、`--muted`）
- [ ] 标题、副标题、正文、脚注的层级清晰，没有字号反转
- [ ] 所有引用、数字、评分、结论都能在 `research.md` 或 sources 附录中找到来源
- [ ] ASCII 架构图页在浏览器 zoom 150% 下仍能清晰阅读
- [ ] 代码页都有语言标签，代码块横向滚动正常，没有被强制换行截断
- [ ] 竞品对比表在 1440 宽屏下不溢出，最右列可完整阅读
- [ ] 最终 HTML 是单文件，可直接本地打开预览

---

## P1 · 建议项

- [ ] 封面主题色与调研对象气质一致，没有明显违和
- [ ] Act 1 偏问题与背景，Act 2 偏架构与原理，Act 3 偏差异与竞品，Act 4 偏 demo 与落地，叙事节奏清楚
- [ ] 同一术语在整份 deck 中前后一致
- [ ] 长段落已拆成短句、列表或卡片，避免整页密字
- [ ] 图片、截图、代码、表格四种信息形态有节奏地交替出现
- [ ] 结束页能自然收束，不像正文被截断

---

## 常见失败信号

- 看到 TODO / TBD / “待补充”
- 架构图退化成段落说明或 Mermaid
- 竞品页只有 feature list，没有统一评分维度
- Act 4 只有 Hello World，没有测试 / 发布 / 调度
- 页面主题切换后，extra layout 颜色没跟着变

任意出现一项，都应回到对应步骤重做，而不是带病交付。
